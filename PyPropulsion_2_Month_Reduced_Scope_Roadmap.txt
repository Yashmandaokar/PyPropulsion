PyPropulsion — Two-Month Reduced-Scope Roadmap
================================================

Project Goal
------------
Build a modular preliminary liquid rocket engine design toolkit in Python.

The tool should take simplified mission-level requirements, estimate the required mission delta-V, recommend a rough launch vehicle architecture, suggest suitable propellant options, and then perform first-order engine sizing, combustion-property estimation using Cantera, nozzle performance analysis, simplified feed-system sizing, and simplified thermal checks.

This is not intended to be a high-fidelity trajectory optimizer or full rocket engine design code. It is intended to be a professional, well-documented, modular engineering portfolio project.


Main User Workflow
------------------
The intended final user workflow is:

1. User enters mission requirements.
2. Mission module estimates required delta-V and recommends number of stages.
3. Propellant-selection module suggests suitable propellant combinations.
4. User selects a propellant pair, or accepts the default recommendation.
5. Engine calculator estimates thrust, mass flow, burn time, propellant split, and propellant mass.
6. Cantera combustion module estimates chamber gas properties.
7. Nozzle module estimates throat area, exit conditions, expansion ratio, and ideal performance.
8. Feed-system module estimates pressure requirements or pump power.
9. Thermal module estimates rough heat flux and wall-temperature risk.
10. Integrated report summarizes the preliminary engine design.


Recommended Reduced-Scope Phases
================================

Phase 0 — Mission Requirement and Architecture Module
-----------------------------------------------------
Purpose:
Convert mission-level requirements into approximate vehicle and engine-level design targets.

Inputs:
- Payload mass [kg]
- Target orbit altitude [m]
- Target inclination [deg]
- Mission type, e.g. LEO, SSO, GTO
- Launch site or launch-site latitude
- Optional user-selected propellant family

Core calculations:
- Circular orbital velocity
- Approximate delta-V budget
- Gravity-loss estimate
- Drag-loss estimate
- Steering/inclination-loss estimate, if simplified
- Total mission delta-V estimate
- Rule-based stage-count recommendation
- Rough thrust-class recommendation

Simplified assumptions:
- No full trajectory simulation
- No launch azimuth optimization
- No detailed aerodynamic loss model
- Losses are estimated using engineering rules of thumb

Example code output:

------------------------

Phase 1 — Rocket Engine Calculator
----------------------------------
Purpose:
Convert engine-level requirements into first-order engine sizing quantities.

Inputs:
- Required thrust [N]
- Specific impulse estimate [s]
- Burn time [s]
- Mixture ratio O/F [-]
- Chamber pressure [Pa]
- Ambient pressure [Pa]
- Number of engines

Core calculations:
- Effective exhaust velocity
- Total mass flow rate
- Oxidizer mass flow rate
- Fuel mass flow rate
- Total propellant mass
- Propellant mass per engine
- Basic performance summary

Simplified assumptions:
- Specific impulse may be user-defined or estimated from propellant selection
- No detailed nozzle calculation yet
- No combustion chemistry yet

Phase 2 — Nozzle Module
-----------------------
Purpose:
Estimate ideal nozzle performance and geometry using quasi-1D compressible-flow relations.

Inputs:
- Chamber pressure [Pa]
- Chamber temperature [K]
- Specific heat ratio gamma [-]
- Gas constant [J/kg/K]
- Mass flow rate [kg/s]
- Ambient pressure [Pa]
- Desired expansion ratio, or exit pressure target

Core calculations:
- Choked-flow throat area
- Throat diameter
- Exit Mach number
- Exit pressure
- Exit temperature
- Exit velocity
- Expansion ratio
- Exit area
- Exit diameter
- Thrust coefficient
- Ideal vacuum and sea-level thrust
- Ideal specific impulse

Simplified assumptions:
- Quasi-1D isentropic flow
- Constant gamma
- No boundary-layer losses
- No nozzle separation model except simple warning
- No bell contour optimization

Phase 3 — Cantera Combustion Module
-----------------------------------
Purpose:
Use Cantera to estimate equilibrium combustion properties for selected propellant pairs.

Recommended first propellant pairs:
- LOX-CH4
- LOX-H2

Optional later surrogate:
- LOX-RP1 approximated using a hydrocarbon surrogate such as dodecane, if mechanism support is available

Inputs:
- Fuel species, e.g. CH4
- Oxidizer species, e.g. O2
- Chamber pressure [Pa]
- Initial reactant temperature [K]
- Equivalence ratio or mixture ratio
- Cantera mechanism file, e.g. gri30.yaml for methane-air/oxygen examples

Core calculations:
- Adiabatic flame temperature using equilibrium HP calculation
- Equilibrium product composition
- Mixture molecular weight
- Specific heat at constant pressure
- Specific heat at constant volume
- Specific heat ratio gamma
- Gas constant of combustion products
- Optional characteristic velocity estimate if implemented

Simplified assumptions:
- Equilibrium combustion only
- No finite-rate flame simulation
- No injector atomization
- No chamber residence-time model
- No combustion instability model

Example code output:

Phase 4 — Simplified Feed-System Module
---------------------------------------
Purpose:
Estimate basic feed-system requirements for delivering propellant to the chamber.

Inputs:
- Chamber pressure [Pa]
- Injector pressure-drop fraction [-]
- Feed-line pressure-drop estimate [Pa]
- Oxidizer mass flow rate [kg/s]
- Fuel mass flow rate [kg/s]
- Oxidizer density [kg/m^3]
- Fuel density [kg/m^3]
- Pump efficiency or pressure-fed tank margin

Core calculations:
- Injector pressure drop
- Required upstream pressure
- Required pump pressure rise
- Oxidizer pump power
- Fuel pump power
- Approximate tank volume
- Approximate propellant volume

Simplified assumptions:
- Steady flow
- No detailed piping network
- No cavitation/NPSH model in first version
- No turbopump turbine design
- No transient tank pressurization

Phase 5 — Simplified Thermal Module
-----------------------------------
Purpose:
Estimate first-order thermal loading and wall-temperature risk.

Inputs:
- Chamber pressure [Pa]
- Chamber temperature [K]
- Gas properties from combustion module
- Throat diameter [m]
- Wall thickness [m]
- Wall thermal conductivity [W/m/K]
- Coolant temperature [K]
- Coolant-side heat-transfer coefficient [W/m^2/K]

Core calculations:
- Approximate gas-side heat-transfer coefficient
- Approximate heat flux near chamber/throat
- 1D wall conduction temperature drop
- Coolant-side heat-transfer estimate
- Rough wall-temperature estimate
- Thermal warning if wall temperature exceeds limit

Possible model:
- Start with simplified 1D thermal-resistance model
- Optionally add Bartz-type heat-transfer estimate later

Simplified assumptions:
- Steady-state thermal model
- No full regenerative cooling-channel network
- No structural stress model
- No material creep/fatigue model
- No transient startup heating

Example code output:

--------------------

Estimated time:
1 week


Phase 6 — Integrated Preliminary Engine Report
----------------------------------------------
Purpose:
Connect all modules and generate one complete preliminary design summary.

Inputs:
- MissionRequirement object
- Optional user-selected propellant
- Optional user-selected engine thrust, chamber pressure, mixture ratio, and burn time
- Default assumptions if values are missing

Core workflow:
1. Analyze mission
2. Estimate delta-V
3. Recommend stage count
4. Suggest propellant options
5. Create preliminary engine requirement
6. Run Phase 1 engine sizing
7. Run Cantera combustion analysis
8. Run nozzle analysis
9. Run feed-system estimate
10. Run thermal estimate
11. Generate final report

What This Project Should Not Include in Two Months
==================================================

Do not include:
- Full 6-DOF trajectory simulation
- Full launch vehicle mass optimization
- Detailed injector atomization model
- Detailed turbopump design
- Full finite-rate combustion simulation
- Combustion instability prediction
- Full regenerative cooling-channel design
- Structural FEM
- CFD coupling
- GUI/web dashboard unless everything else is already stable

These can be listed as future work.
